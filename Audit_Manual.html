1. Overview & Security

The Purchase Register Audit Tool is a browser-based application designed for Internal Auditors of Indian Private Limited companies. It performs automated compliance checks on purchase register data without requiring any server, internet connection, or data upload.

✓ Key Security Feature: All data processing occurs entirely within your browser's memory. No purchase data, supplier names, rates, or quantities are transmitted over the internet or stored on any remote server. This makes the tool compliant with strict internal audit confidentiality requirements.

What This Tool Does

  • Accepts Excel (.xlsx) or CSV (.csv) purchase register files
  • Runs 25 mandatory audit checks across 4 tiers
  • Detects date sequence violations, quantity mismatches, process control failures, and fraud indicators
  • Calculates estimated financial impact of discrepancies
  • Generates a professional, filterable dashboard with severity-coded findings
  • Exports complete audit reports as formatted Excel files

What You Need

  • A Windows/Mac/Linux laptop with any modern web browser (Chrome, Edge, Firefox, Safari)
  • The audit tool HTML file
  • (Optional) The SheetJS library file for Excel support in offline mode
  • Your company's purchase register data in Excel or CSV format

3. Installation Methods

Choose the method that best fits your organization's IT policies and offline requirements.

Method A: Online-First Setup (Easiest)

Internet Required Once

1

Download the Audit Tool

Save the PurchaseRegisterAuditTool.html file to a folder on your computer, e.g., C:\AuditTools\ or Documents/Audit/.

2

First Launch with Internet

Double-click the HTML file to open it in your browser while connected to the internet. The tool will automatically load the SheetJS library from CDN for Excel parsing support.

3

Verify It Works

Upload one of the sample CSV files to confirm the dashboard loads. Once verified, the tool will continue to work offline for CSV files indefinitely.

4

For Permanent Offline Excel Support

Download the SheetJS library file from:
https://cdn.sheetjs.com/xlsx-0.20.1/package/dist/xlsx.full.min.js
Save it in the same folder as the HTML file, then edit line ~12 of the HTML to change:
<script src="https://cdn.sheetjs.com/xlsx-0.20.1/package/dist/xlsx.full.min.js"></script>
to:
<script src="xlsx.full.min.js"></script>

Method B: Fully Offline Setup (Air-Gapped Systems)

No Internet Ever

1

Prepare on an Internet-Connected Machine

On a machine with internet access, download both files:

PurchaseRegisterAuditTool.html
xlsx.full.min.js (from https://cdn.sheetjs.com/xlsx-0.20.1/package/dist/xlsx.full.min.js)
2

Modify the HTML for Local SheetJS

Open PurchaseRegisterAuditTool.html in any text editor (Notepad, VS Code, TextEdit). Find this line near the top:

<script src="https://cdn.sheetjs.com/xlsx-0.20.1/package/dist/xlsx.full.min.js"></script>

Replace it with:

<script src="xlsx.full.min.js"></script>
3

Transfer to Air-Gapped Laptop

Copy both files to a USB drive. Transfer them to the target laptop's folder (e.g., C:\AuditTools\). Both files must remain in the same folder.

4

Verify & Test

Double-click the HTML file on the air-gapped machine. Test with a CSV file first, then an Excel file to confirm SheetJS is loading from the local copy.

Step 2: Upload the File

1

Click Select File or Drag & Drop

On the audit tool homepage, click the Select File button and choose your .xlsx or .csv file. Alternatively, drag the file from File Explorer and drop it onto the upload area.

2

Wait for Processing

A loading spinner will appear while the tool:

  • Parses the file (Excel via SheetJS, CSV via native parser)
  • Normalizes dates and numbers
  • Runs all 25 audit checks across 4 tiers
  • Calculates financial impacts and systemic observations

Processing time: ~1-3 seconds for 1,000 rows; ~10-20 seconds for 50,000 rows.

Step 3: Review the Dashboard

After processing, the dashboard displays six metric cards:

  • Total Records: Number of rows processed
  • Total Flags: Sum of all findings across all tiers
  • High Risk: Findings requiring immediate auditor action
  • Medium Risk: Findings requiring review and documentation
  • Low Risk: Minor observations for file note
  • Est. Financial Impact: Cumulative overbilling from conversion factor discrepancies

Step 4: Navigate Tier Tabs

Click the tier tabs to view categorized findings:

  • Tier 1: Date Sequence & Quantity Reconciliation
  • Tier 2: Process Control failures
  • Tier 3: Fraud Indicators
  • Tier 4: Systemic Observations (summary + approver concentration table)
  • All Findings: Combined view with tier labels

Step 5: Filter & Investigate

Each tier table has two filters:

  • Text Filter: Type keywords to filter by supplier name, item code, flag reason, etc.
  • Severity Filter: Dropdown to show only High, Medium, or Low risk findings

Each flagged row shows: Row Number, Audit Check Name, Flag Reason, Severity badge, and Key Values involved.

7. Understanding Audit Results

Severity Color Coding

BadgeMeaningAuditor Action
HIGHCritical violation or fraud indicatorImmediate investigation, document evidence, escalate to management
MEDIUMProcess control failure or compliance gapReview supporting documents, verify approvals, note in audit working papers
LOWMinor observation or data quality issueNote for process improvement, follow up in next audit cycle

How to Read a Flagged Row

Row: 42 | Check: 1. Bill before GRN
Reason: Party Bill Date (12/04/2025) is before GRN Date (15/04/2025)
Severity: HIGH
Columns: Party Bill Date, GRN Date
Values: Bill: 12/04/2025, GRN: 15/04/2025

Interpretation: The supplier raised a bill before goods were actually received. This could indicate:

  • Back-dated GRN entry (process failure)
  • Advance billing without goods receipt (cash flow risk)
  • Fraudulent early payment pressure

Financial Impact Calculation

The Est. Financial Impact card shows the cumulative overbilling detected from Conversion Factor discrepancies (Check 17). The formula used is:

Impact = |Max Difference among Conv Fact, Item Conversion, PO Item Conversion| × Basic Rate × AQTY

This is an estimated figure for audit scoping. Exact overbilling requires verification against actual invoices and UOM master data.

8. Audit Tiers Explained

Tier 1 — Date Sequence & Quantity Reconciliation

Focuses on chronological logic and physical quantity matching. These are foundational checks that verify the basic integrity of the purchase-to-pay cycle.

CheckWhat It DetectsTypical Root Cause
1. Bill before GRNInvoice dated before goods receiptBack-dated GRN, advance billing
2. Voucher before GRNAccounting entry before physical receiptPre-accounting, system backdating
3. Approval before InspectionQuality approval before QC checkProcess bypass, rubber-stamp workflow
4. Uninspected AcceptanceGoods accepted without inspection dateMissing QC step, undocumented waiver
5. Quantity MismatchChallan ≠ Accepted ≠ ReachedShort receipt, damage, pilferage
6. Rate Variance >5%Same item/supplier with fluctuating ratesPrice manipulation, wrong item coded
7. Late DeliveryGRN after Order Due DateSupplier delay, poor planning

Tier 2 — Process Control

Validates whether procurement controls (indenting, ordering, approval limits, QC) were followed.

CheckWhat It DetectsTypical Root Cause
8. No Indent NumberGRN without purchase requisitionUnauthorized purchase, emergency buy
9. No Order NumberGRN without purchase orderProcurement bypass, verbal orders
10. Approval ConcentrationOne person approves >40% of valueSegregation of duties failure
11. Rubber Stamp QCBlank QC remark but full acceptanceFake QC, inspector negligence
12. Blank Batch NumberMissing batch traceabilityData entry skip, non-batch item
13. Master Data BreachSupplier code/name mismatchIncomplete master, data corruption

Tier 3 — Fraud Indicators

Pattern-based checks that identify behaviors commonly associated with procurement fraud.

CheckWhat It DetectsTypical Root Cause
14. Split OrdersSame supplier/item within 7 days, different POsOrder splitting to avoid approval limits
15. Transporter=SupplierCarrier name matches a supplier nameRelated-party transport, inflated freight
16. Vehicle/LR MismatchInconsistent vehicle identificationDiversion of goods, ghost deliveries
17. Conversion Factor MismatchUOM conversion factors differOverbilling through UOM manipulation
18. Godown-Division MismatchSame item in multiple godowns, one divisionCost misallocation, inventory fraud
19. Gate Entry Gaps/DuplicatesMissing or duplicate gate entry numbersMissing entries, duplicate billing

Tier 4 — Systemic Observations

Aggregate statistics that reveal organizational patterns rather than individual transaction issues.

  • % Blank QC Remarks: Indicates QC culture weakness
  • % Missing Indent Linkage: Shows procurement discipline
  • Top 5 Suppliers by Value: Concentration risk analysis
  • Approval Concentration Ratio: Segregation of duties health
  • Total Overbilling Estimate: Financial quantification of Check 17
  • Bills Before GRN Count: Frequency of timing violations

9. Exporting Reports

Generating the Audit Report

  1. After processing, click the Export Full Report (Excel) button on the dashboard.
  2. The tool generates a multi-sheet Excel workbook containing:
Sheet NameContents
All FindingsEvery flagged row with tier, check, reason, severity, and values
Systemic SummaryAggregate metrics (QC blank %, indent missing %, overbilling, etc.)
Top 5 SuppliersRanked list of suppliers by transaction value
Approver ConcentrationEach approver's count, value, percentage, and limit status

Export Format Behavior

With SheetJS (online or offline): Exports as a proper .xlsx file with multiple formatted sheets.
Without SheetJS (CSV-only mode): Falls back to a single .csv file containing all findings. Systemic summaries must be copied manually from the dashboard.

Report Naming

Exported files are automatically named:

Purchase_Register_Audit_Report_YYYY-MM-DD.xlsx

Where the date is the current system date. This ensures version control when multiple audits are performed.

11. Troubleshooting

Common Issues & Solutions

IssueCauseSolution
"Error processing file" Corrupted Excel file or unsupported format Save the file as CSV (Save As → CSV UTF-8) and retry. CSV is the most reliable format.
Excel file not parsing (SheetJS not loaded) Offline mode without local SheetJS file Convert file to CSV, or download xlsx.full.min.js and place it next to the HTML file.
Dates showing as numbers (e.g., 45321) Excel serial dates not recognized The tool auto-detects Excel serial dates. If issues persist, format dates as DD/MM/YYYY in your source file before export.
Very slow processing (>60 seconds) File too large (>100,000 rows) or low RAM Split the register into quarterly files. Close other browser tabs. Use Chrome for best performance.
Browser shows blank page JavaScript disabled or file path issue Enable JavaScript in browser settings. Ensure the file is opened via double-click, not from a zip archive.
Export button does nothing SheetJS missing or popup blocker Check for popup blockers. If using CSV fallback, the file auto-downloads. Check your Downloads folder.
Columns not recognized Column names don't match expected headers Rename columns in your source file to match the expected names (case-insensitive match is supported, but special characters/spaces matter).
All rows flagged as "High Risk" incorrectly Date format misinterpretation Ensure dates are in DD/MM/YYYY format. MM/DD/YYYY (US format) may cause false flags.

Performance Tips

  • Chrome/Edge perform 2-3x faster than Firefox for large files due to V8 engine optimization
  • Close unnecessary browser tabs to free memory
  • For files >50 MB, consider splitting by quarter or division
  • CSV files process faster than Excel files

Getting Help

Since this tool is fully offline and client-side:

  • There is no "server" to check — all issues are local to your file or browser
  • Test with the provided sample files first to isolate whether the issue is with your data or the tool
  • If a specific row causes errors, check that row for unusual characters (tabs, newlines, special symbols) in text fields

12. Data Security Protocols

What the Tool Does NOT Do

  • ❌ Does not connect to the internet after page load
  • ❌ Does not upload data to any server or cloud service
  • ❌ Does not store cookies or localStorage data
  • ❌ Does not retain data after browser refresh (all in-memory only)
  • ❌ Does not modify your source Excel/CSV file

What the Tool DOES Do

  • ✅ Reads your file into browser memory (RAM) only
  • ✅ Processes entirely within JavaScript engine
  • ✅ Generates export files to your local Downloads folder only
  • ✅ Clears all data when you close the tab or click "New Audit"

Auditor Responsibilities

Your duty to protect data:
  • Delete exported reports from the Downloads folder after copying to your secure audit working papers folder
  • Do not leave the browser open with sensitive data when away from your desk
  • Ensure the laptop has disk encryption (BitLocker/FileVault) enabled
  • When using the tool on a shared computer, clear browser cache after use (though the tool doesn't cache data, browser may cache the HTML file)

Compliance Notes

This tool is designed for use under the Companies Act, 2013 (India) internal audit framework and supports:

  • Section 134(5) — Internal financial controls
  • Section 138 — Internal audit function requirements
  • SA 240 (Standards on Auditing) — Fraud consideration
  • GST Input Tax Credit reconciliation support (through date/quantity verification)
